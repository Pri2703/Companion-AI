## 3. Modularization (Monolith to Backend Core)
* Dissected the monolithic `detect_depth_tts_save.py` and modularized it:
  * **`backend/core/detection_engine.py`:** Initializes the PyTorch hub model (`intel-isl/MiDaS`) and local YOLOv8 weights (`models/best.pt`). Emits an array of structured JSON dicts (label, confidence, bbox, depth mapped to meters).
  * **`backend/core/alert_engine.py`:** Consumes the detected JSON array. Runs mathematical center checks to determine direction (`on left`, `on right`, `ahead`) and computes urgency mapping based on proximity.
* **TTS Removal:** Completely removed `pyttsx3`, `gTTS`, and threaded `pyttsx3` locks so the frontend browser (e.g., React Speech Synthesis) can handle voice native rendering. Returned simple strings inside a JSON list instead.

## 4. FastAPI Orchestrator implementation
* Replaced the main execution script entirely with **`backend/app.py`**.
* Implemented threading: A background `camera_loop(source=0)` reads the camera continuously to avoid IO blocking the web server endpoints.
* **Endpoints exposed:**
  * `GET /video`: Streams the annotated frames natively as MJPEG (`multipart/x-mixed-replace`).
  * `GET /data`: Serves the live JSON payload resulting from the AI engines. Structure: `{"objects": [...], "alerts": [...]}`.

## 5. Dependency Repair
* Generated a strict `requirements.txt` dropping dependencies like `pyttsx3`.
* Included `timm` explicitly, satisfying implicit missing module errors thrown internally by the `MiDaS` Torch hub loader executing BEiT backbones.
* Included `fastapi` and `uvicorn`.
